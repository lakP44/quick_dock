# Quick Dock

Rust + egui/eframe로 만든 작은 도킹 런처입니다. 화면 가장자리에 붙여 두고 복사, 실행, 파일/폴더 열기 작업을 빠르게 누르는 용도입니다.

## 기능

- 화면 왼쪽, 오른쪽, 위, 아래 가장자리에 도킹
- 마우스를 올리면 확장, 벗어나면 접힘
- 탭별 작업 관리
- 텍스트 클립보드 복사 (날짜·클립보드·입력값 등 템플릿 변수 지원)
- 프로그램 실행 / 관리자 권한 실행
- 파일/폴더 열기
- 앱 안에서 설정 편집
- 실행/열기 경로 검증, 테스트 실행, 복사 테스트
- 설정 저장 전 `env\quick_dock.ini.bak` 자동 백업
- 작업 버튼 우클릭 메뉴 (편집/복제/순서 이동/삭제/경로 복사 등)
- 검색 팔레트(`Ctrl+Space` 또는 `검색` 버튼)로 모든 탭의 작업을 빠르게 실행
- 시스템 트레이 아이콘 (열기/설정/종료, 시작 시 자동 실행 토글)
- 두 번째 실행 시 기존 창을 앞으로 가져오기
- 모니터별 도킹 위치·크기 기억, 창모드 크기 별도 저장
- 동작 기록 로그 `env\quick_dock.log`

## 빌드

```bat
cargo build --release
```

빌드 결과:

```text
target\release\quick_dock.exe
```

배포용 빌드는 다음 스크립트를 사용할 수 있습니다.

```bat
build_windows_release.bat
```

이 스크립트는 `env` 폴더의 파일을 `target\release\env`로 복사하되, 이미 존재하는 사용자 설정 파일은 덮어쓰지 않습니다.

## 설정 파일

Quick Dock은 실행 파일 옆의 `env\quick_dock.ini`를 사용합니다. 앱의 설정 화면에서 편집하는 것을 권장합니다.

```ini
[layout]
schema_version=1
expanded_width=350
expanded_height=430

[tab.1]
name=기본

[tab.1.item.1]
kind=copy_text
name=Jira - 검토 완료
text=검토 완료했습니다.\n\n확인 내용:\n- \n

[tab.1.item.2]
kind=run_app
name=메모장 실행
command=notepad.exe
arguments=

[tab.1.item.3]
kind=open_path
name=다운로드 폴더 열기
path=C:\Users\%USERNAME%\Downloads
```

`arguments`는 `값1|값2|값3`처럼 `|`로 구분합니다. 줄바꿈은 `\n`으로 저장됩니다.

## 복사 템플릿 변수

복사(`copy_text`) 항목의 내용에는 다음 변수를 넣을 수 있습니다. 복사하는 순간 실제 값으로 치환됩니다.

| 변수 | 치환 내용 |
|---|---|
| `{date}` | 오늘 날짜 `YYYY-MM-DD` |
| `{time}` | 현재 시간 `HH:MM:SS` |
| `{datetime}` | `YYYY-MM-DD HH:MM:SS` |
| `{clipboard}` | 현재 클립보드 텍스트 |
| `{selection}` | 직전에 사용하던 창에서 선택된 텍스트 (Windows, best-effort) |
| `{input:라벨}` | 복사 직전 작은 입력창을 띄워 받은 값. 같은 라벨은 한 번만 묻습니다 |

예: `작업 완료했습니다. 티켓: {input:Jira 번호} ({datetime})` 을 누르면 입력창이 떠 번호를 받고 복사합니다.

## 검색 팔레트

`Ctrl+Space` 또는 헤더의 `검색` 버튼으로 팔레트를 엽니다.

- 작업 이름·복사 내용·탭 이름으로 검색
- `↑` `↓`로 이동, `Enter`로 실행, `Esc`로 닫기
- 검색어가 비어 있으면 최근 실행 항목(`★`)을 먼저 보여줍니다

## 작업 버튼 우클릭 메뉴

작업 버튼을 우클릭하면 설정 화면에 들어가지 않고도 가볍게 관리할 수 있습니다.

- 공통: 편집 / 복제 / 위로·아래로 / 삭제
- 복사 항목: 내용 보기 / 복사
- 실행 항목: 실행 / 관리자 권한으로 실행 / 파일 위치 열기 / 경로 복사
- 열기 항목: 열기 / 탐색기에서 선택 / 경로 복사

순서 이동·복제·삭제는 즉시 `quick_dock.ini`에 저장됩니다.

## 트레이 아이콘과 자동 실행

- 트레이 아이콘 좌클릭: 창을 앞으로 가져오기
- 트레이 아이콘 우클릭 메뉴: `열기` / `설정` / `Windows 시작 시 실행`(체크) / `종료`
- 헤더의 `자동시작` 버튼으로도 시작 프로그램 등록을 켜고 끌 수 있습니다 (레지스트리 `HKCU\...\Run`)
- 이미 실행 중일 때 다시 실행하면 새 창을 띄우지 않고 기존 창을 앞으로 가져옵니다

## 로그

동작과 오류는 `env\quick_dock.log`에 시간과 함께 기록됩니다. 실행이 안 될 때 원인(파일 없음, 권한, 형식 등)을 여기서 확인할 수 있습니다.

## 조작

- 상단 `Quick Dock` 제목줄 드래그: 창 이동 또는 가장자리 도킹
- `검색`(또는 `Ctrl+Space`): 검색 팔레트 열기
- `새 탭`: 탭 추가
- `자동시작`: Windows 시작 시 자동 실행 토글
- `설정`: 현재 탭 작업 편집
- 설정 저장: 기존 `quick_dock.ini`를 `quick_dock.ini.bak`로 백업한 뒤 저장
- 실행 항목 `찾아보기...`: exe 파일 선택
- 실행/열기 항목 `테스트`: 저장 전 실제 동작 확인
- 확장된 창의 네 모서리 드래그: 창 크기 조절 (도킹/창모드 크기는 따로 저장)
